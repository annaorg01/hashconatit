# 📖 מדריך הפרויקט — השכונתית

מערכת לאיסוף לידים ממכונות אוטומטיות (דרך קוד QR) ולשליחת קמפיינים שיווקיים
ב-SMS, אימייל ו-WhatsApp. בנויה ב-**Node.js**, עם **Google Sheets** כמסד נתונים.
מותאמת למובייל ולחוק הספאם הישראלי.

---

## 📂 איפה הפרויקט נמצא

כל הקבצים נמצאים בתיקייה בשם:

```
hashchunatit-app/
```

זו התיקייה שקיבלת בתיקיית הפלט שלי. כשתפתחי אותה במחשב, מבנה הקבצים הוא כך:

```
hashchunatit-app/
│
├── מדריך-הפרויקט.md         ← הקובץ הזה (ההסבר בעברית)
├── מבנה-גוגל-שיטס.md        ← מבנה הגיליון (הטאבים והעמודות)
├── README.md                ← הוראות מפורטות באנגלית
├── package.json             ← הגדרות הפרויקט והתלויות
├── .env.example             ← תבנית הגדרות (להעתיק ל-.env)
├── render.yaml              ← הגדרות פריסה ל-Render
├── start.command            ← הפעלה מקומית בלחיצה כפולה (Mac)
├── .gitignore
│
├── public/                  ← כל מה שהמשתמש והמנהל רואים בדפדפן
│   ├── index.html           ← דף הנחיתה (הטופס שנפתח מסריקת ה-QR)
│   ├── login.html           ← מסך כניסת מנהל
│   └── admin.html           ← לוח הבקרה (מנויים + שליחת קמפיינים)
│
└── src/                     ← קוד השרת
    ├── server.js            ← השרת הראשי + כל ה-API
    ├── sheets.js            ← שכבת הנתונים מול Google Sheets (ה-API)
    ├── auth.js              ← אבטחת כניסת המנהל
    └── services/
        ├── messaging.js     ← שליחת SMS / WhatsApp / אימייל

```

> 💡 הנתונים (הלידים) נשמרים ב-Google Sheets — לא במחשב. התיקייה `uploads/`
> נוצרת לבד בפעם הראשונה ושומרת את קובצי המדיה של קמפיינים.

## ▶️ איך להריץ את הפרויקט (שלב אחר שלב)

### דרישה מוקדמת
צריך שיהיה מותקן **Node.js גרסה 18 ומעלה**. לבדיקה: בטרמינל כותבים `node -v`.

### שלב 1 — הכנת ה-Google Sheets
לפי הקובץ `מבנה-גוגל-שיטס.md`, יוצרים גיליון עם שני טאבים (`Leads` ו-`Campaigns`)
והכותרות המתאימות. שומרים את ה-`SHEETS_API_KEY` שקיבלת מ-sheets-connector.

### שלב 2 — להיכנס לתיקייה
```bash
cd hashchunatit-app
```

### שלב 3 — להתקין את התלויות
```bash
npm install
```

### שלב 4 — להגדיר את ההגדרות
```bash
cp .env.example .env
```

פותחים את `.env` בעורך טקסט וממלאים לפחות:
- `SHEETS_API_KEY` — המפתח של ה-API ל-Google Sheets (חובה!)
- `ADMIN_PASSWORD` — סיסמת המנהל שלך (חובה לשנות!)
- `SESSION_SECRET` — מחרוזת אקראית ארוכה לאבטחה
- מפתחות הספקים (Twilio / SMTP) — רק לערוצים שתרצי להפעיל בפועל

### שלב 5 — להריץ
```bash
npm start
```

או, ב-Mac, פשוט **לחיצה כפולה על `start.command`** (מתקין, מריץ ופותח דפדפן לבד).

יופיע:
```
🟢 השכונתית running → http://localhost:3000
```

## 🌐 איך פותחים את הדפים

| מה | כתובת |
|----|--------|
| **דף הנחיתה** (הטופס מה-QR) | `http://localhost:3000/?machine=building_A` |
| **לוח הבקרה למנהל** | `http://localhost:3000/admin` |

הכניסה ללוח הבקרה היא עם שם המשתמש והסיסמה שהגדרת ב-`.env`.

---

## 📱 קודי ה-QR למכונות

מייצרים קוד QR לכל מכונה/בניין, שמפנה לכתובת שלך עם פרמטר `machine` שונה:

```
https://your-domain.co.il/?machine=building_A
https://your-domain.co.il/?machine=building_B
```

ערך ה-`machine` נשמר אצל כל ליד, כך שתמיד תדעי מאיזו מכונה הגיע כל לקוח.

## 📨 חיבור ערוצי השליחה

| ערוץ | ספק | מה למלא ב-.env |
|------|-----|----------------|
| SMS + WhatsApp | Twilio | `TWILIO_*` |
| אימייל | כל שרת SMTP (Gmail / SendGrid / Mailgun) | `SMTP_*` |
| סנכרון נוסף לפלטפורמות (Smoove / Inwise / ActiveTrail) | Make.com | `MAKE_WEBHOOK_URL` |

ערוץ שלא הוגדר פשוט מדולג — המערכת לא תיפול בגללו.

## ❓ שאלות נפוצות

**הדף לא נטען / שגיאה בהרצה** — לרוב חסר `npm install`, או שלא מולא `SHEETS_API_KEY` ב-`.env`.

**שכחתי סיסמת מנהל** — היא נמצאת בקובץ `.env` תחת `ADMIN_PASSWORD`.

**איפה נשמרים הלידים?** — ב-Google Sheets שלך (טאב `Leads`). אפשר גם לייצא ל-Excel/CSV
מלוח הבקרה (כפתור "ייצוא ל-CSV/Excel").

הסבר טכני מלא יותר נמצא בקובץ `README.md`, ומבנה הגיליון בקובץ `מבנה-גוגל-שיטס.md`.
