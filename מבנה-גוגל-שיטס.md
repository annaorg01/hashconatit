# 📊 מבנה ה-Google Sheets — השכונתית

המערכת שומרת את הנתונים ב-Google Sheets דרך ה-API של sheets-connector.
צריך **גיליון אחד** עם **שלושה טאבים (לשוניות)**: `Leads` (לידים), `Campaigns` (קמפיינים),
ו-`Promotions` (מבצעים שמוצגים ללקוח).

> ⚠️ חשוב: שמות הכותרות חייבים להיות **בדיוק** כפי שמופיע כאן, באנגלית, בשורה הראשונה
> של כל טאב. הסדר לא חייב להיות זהה, אבל השמות כן. כל ערך נשמר כטקסט.

---

## 🗂️ טאב 1 — `Leads` (לידים)

שם הטאב צריך להתאים ל-`SHEETS_LEADS_TABLE` בקובץ `.env` (ברירת מחדל: `Leads`).

שורת הכותרות (שורה 1), משמאל לימין:

| עמודה | כותרת (בדיוק כך) | תיאור | דוגמה |
|------:|-------------------|-------|-------|
| A | `id` | מזהה ייחודי — **מספר הטלפון** (גם מונע כפילויות) | `0501234567` |
| B | `full_name` | שם מלא | `דנה כהן` |
| C | `phone` | מספר טלפון (10 ספרות) | `0501234567` |
| D | `email` | אימייל (יכול להיות ריק) | `dana@example.com` |
| E | `machine_id` | מזהה המכונה מה-QR (`?machine=`) | `building_A` |
| F | `consent_granted` | אישור דיוור: `1` = אישר, `0` = לא | `1` |
| G | `status` | סטטוס הליד ב-CRM | `חדש` / `פעיל` / `VIP` |
| H | `tags` | תגיות מופרדות בפסיקים | `מבניין א,אוהב קפה` |
| I | `notes` | הערות חופשיות של הצוות | `ביקש עדכון על מבצעי בוקר` |
| J | `created_at` | חותמת זמן ההרשמה | `2026-06-13 14:32:05` |

**שורת הכותרות להעתקה ישירה:**
```
id	full_name	phone	email	machine_id	consent_granted	status	tags	notes	created_at
```

> העמודות `status`, `tags`, `notes` מנוהלות מצד האדמין (ה-CRM). ליד חדש מתחיל
> בסטטוס `חדש` אוטומטית.

## 🗂️ טאב 2 — `Campaigns` (קמפיינים)

שם הטאב צריך להתאים ל-`SHEETS_CAMPAIGNS_TABLE` בקובץ `.env` (ברירת מחדל: `Campaigns`).
טאב זה מתמלא אוטומטית בכל שליחה — לא צריך למלא ידנית, רק ליצור את הכותרות.

| עמודה | כותרת (בדיוק כך) | תיאור |
|------:|-------------------|-------|
| A | `id` | מזהה הקמפיין (חותמת זמן) |
| B | `channels` | הערוצים שנבחרו, כ-JSON, למשל `["sms","email"]` |
| C | `audience` | קהל היעד, כ-JSON |
| D | `body` | תוכן ההודעה |
| E | `media_path` | שם קובץ המדיה שצורף (אם יש) |
| F | `scheduled_at` | מועד שליחה מתוזמן (ריק = נשלח מיד) |
| G | `status` | `pending` / `scheduled` / `sent` / `failed` |
| H | `recipients` | כמה נמענים בקהל היעד |
| I | `sent_count` | לכמה נשלח בפועל |
| J | `result_log` | פירוט תוצאות כ-JSON |
| K | `created_at` | חותמת זמן יצירת הקמפיין |

**שורת הכותרות להעתקה ישירה:**
```
id	channels	audience	body	media_path	scheduled_at	status	recipients	sent_count	result_log	created_at
```

## 🗂️ טאב 3 — `Promotions` (מבצעים)

שם הטאב צריך להתאים ל-`SHEETS_PROMOTIONS_TABLE` בקובץ `.env` (ברירת מחדל: `Promotions`).
אלו המבצעים שהלקוח רואה במסך הראשי. מנוהל כולו מצד האדמין (אפשר ליצור/לערוך/להפעיל/לכבות),
אבל אפשר גם לערוך ידנית בגיליון.

| עמודה | כותרת (בדיוק כך) | תיאור | דוגמה |
|------:|-------------------|-------|-------|
| A | `id` | מזהה ייחודי (חותמת זמן, נוצר אוטומטית) | `1781332616416` |
| B | `title` | כותרת המבצע | `1+1 על כל המשקאות` |
| C | `description` | תיאור קצר | `בכל קנייה מעל 10 ₪` |
| D | `badge` | תווית קטנה (לא חובה) | `חדש` / `מוגבל` |
| E | `emoji` | אייקון לתצוגה | `🥤` |
| F | `active` | מוצג ללקוח? `1` = כן, `0` = מוסתר | `1` |
| G | `sort_order` | סדר הצגה (מספר קטן = ראשון) | `1` |
| H | `created_at` | חותמת זמן | `2026-06-13 14:32:05` |

**שורת הכותרות להעתקה ישירה:**
```
id	title	description	badge	emoji	active	sort_order	created_at
```

## 🔑 חיבור ל-.env

אחרי שיצרת את הגיליון והטאבים, מלאי בקובץ `.env`:

```
SHEETS_API_BASE=https://sheets-connector.vercel.app/api/v1
SHEETS_PROJECT_ID=c1014db5-9794-41ef-8887-3aae4bdb9c71
SHEETS_API_KEY=<<המפתח שקיבלת בעת שמירת החיבור>>
SHEETS_LEADS_TABLE=Leads
SHEETS_CAMPAIGNS_TABLE=Campaigns
SHEETS_PROMOTIONS_TABLE=Promotions
```

> אם שמות הטאבים שלך שונים (למשל `RAIL1`), פשוט עדכני את
> `SHEETS_LEADS_TABLE` / `SHEETS_CAMPAIGNS_TABLE` / `SHEETS_PROMOTIONS_TABLE` בהתאם.

## 🔄 איך המערכת משתמשת ב-API

| פעולה במערכת | קריאה ל-API |
|--------------|-------------|
| ליד חדש נרשם | `POST /tables/Leads` (או `PATCH` אם הטלפון כבר קיים — עדכון במקום כפילות) |
| צפייה/ייצוא מנויים | `GET /tables/Leads?limit&offset` |
| שליחת קמפיין | קורא לידים מאושרים ב-`GET`, יוצר רשומה ב-`POST /tables/Campaigns` |

המערכת שולפת רק לידים עם `consent_granted = 1` לשליחות — כך נשמרת עמידה בחוק הספאם
גם כשהנתונים בגיליון.
